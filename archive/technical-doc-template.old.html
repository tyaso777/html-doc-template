1. 概要

このページでは、PyodideによりPythonをブラウザ上で実行します。 サーバー側のPython環境は不要です。ただし、初回読み込み時にPyodide本体を読み込むため、少し時間がかかります。

補足Note（aside.callout-success）は、読者に安心して読んでもらうための補足、前提条件、成功状態、推奨事項に使います。 例: このテンプレートはスマートフォンでも読みやすいレイアウトを想定しています。
注意Note（aside.callout-warning）は、注意点、制約、環境依存、事前に知っておくべき例外に使います。 例: 目次項目の自動生成、Python実行、Mermaid、MathJax、PrismはJavaScriptに依存します。

2. テンプレートの使い方

この章は、テンプレートを複製して新しいHTMLドキュメントを作るときの最小限の編集ガイドです。 人間が読んでも、生成AIがHTMLを追記しても、既存の構造に沿いやすいように主要な書き方を示します。

章として目次に出したい範囲は <section id="..." data-toc data-toc-title="..."> で作ります。 小見出しも目次に出したい場合は、見出しに id、data-toc、data-toc-level="3" を付けます。

よく使う部品

  • 補足や推奨事項: <aside class="callout callout-success">
  • 注意点や制約: <aside class="callout callout-warning">
  • 本文中の標準カード: <div class="card">
  • 控えめなカード: <div class="card card-muted">
  • 余白を抑えたカード: <div class="card card-compact">
  • コード表示: <pre class="code-block language-python"><code class="language-python">
  • 数式: インラインは \(...\)、別行表示は \[...\]
  • Mermaid図: <div class="mermaid"> に図の定義を書きます。
目次、Python実行、Mermaid、MathJax、PrismはJavaScriptで拡張されます。 JavaScriptが止まっても本文を読めるように、重要な説明は通常のHTML本文として書いてください。

3. 数式の表示例

MathJaxによるインライン数式と別行数式の表示サンプルです。 新しいドキュメントを作るときは、この章を複製して内容を差し替えると、表示形式を保ったまま編集できます。

インライン数式

本文中に短い数式を入れる場合は \(...\) を使います。 たとえば、正規分布は平均 \(\mu\) と分散 \(\sigma^2\) を使って \(X \sim \mathcal{N}(\mu, \sigma^2)\) と書けます。 標準化した値は \(z = (x - \mu) / \sigma\) です。

別行数式

独立した数式として見せたい場合は \[...\] で囲み、必要に応じて div.math-block に入れます。

\[ f(x) = \frac{1}{\sigma\sqrt{2\pi}} \exp\left(-\frac{(x - \mu)^2}{2\sigma^2}\right) \]
\[ \mathbb{E}[X] = \mu, \qquad \mathrm{Var}(X) = \sigma^2, \qquad \int_{-\infty}^{\infty} f(x)\,dx = 1 \]

4. Pythonコードの表示と実行例

Pythonコードは、読むための静的なコードブロックと、ブラウザ上で実行できるコードエディタを使い分けます。 説明だけでよい場合はコード表示、学習者に試してほしい場合は実行パネルを使います。

Pythonコードの表示例

実行させる必要がないコードは、pre.code-block と code.language-python を組み合わせて表示します。 コードの上には div.code-caption を置くと、言語名やファイル名を示せます。

python
def summarize_scores(scores: list[int]) -> dict[str, float]:
    """Returns simple summary statistics for a score list."""
    total = sum(scores)
    count = len(scores)

    return {
        "count": count,
        "average": total / count,
        "maximum": max(scores),
    }


scores = [82, 90, 76, 88]
summary = summarize_scores(scores)
print(summary)

実行コードの表示例

学習者が編集・実行するコードは、次の実行パネルに表示されます。 初期コードはHTML下部の DEFAULT_CODE を編集して差し替えます。

Python実行機能は、Load Python Runtime を押したときにPyodideをCDNから読み込みます。 PyodideはWeb Worker内の importScripts() で読み込むため、通常の integrity によるSRI検証は使えません。 厳密な供給網管理が必要な資料では、レビュー済みのPyodide一式をローカル配布する構成に変更してください。

ハイライト付きのPythonエディタでコードを編集してから Run Python を押してください。 モバイル環境や低速な回線では、初回のPython runtime読み込みに時間がかかることがあります。

5. 他の言語の表示

          

            Python以外のコードも、Prismの言語クラスを切り替えることで同じデザインで表示できます。
            このテンプレートでは、PowerShellとSQLのPrismコンポーネントも初期状態で読み込んでいます。
          


          
PowerShellコードの表示例

          
powershell

          
Get-ChildItem -Path . -Filter "*.html" |
    Select-Object Name, Length, LastWriteTime |
    Sort-Object LastWriteTime -Descending



          
SQLコードの表示例

          
sql

          
SELECT
    document_id,
    title,
    updated_at
FROM documents
WHERE status = "published"
ORDER BY updated_at DESC;



          

            cmd、bash、JSONなど別の言語を追加したい場合は、対応するPrism言語コンポーネントを読み込み、
            language-... のクラス名をその言語に合わせて変更してください。
            CDNから追加する場合は、追加したファイルにもSRIを設定してください。
          

        


        

          
6. Mermaidでの図の表示

          

            Mermaidは、テキストで書いた定義からフローチャートや構造図を描画するためのライブラリです。
            図を画像として作り直さずに、HTML内のテキストを編集するだけで更新できます。
            フローチャートのほか、シーケンス図、クラス図、状態遷移図なども記述できます。
          


          
mermaid

          
flowchart LR
    A[資料の目的を決める] --> B[章立てを決める]
    B --> C[本文を書く]
    B --> D[演習やコード例を作る]
    C --> E[図や数式を追加する]
    D --> F[実行例を確認する]
    E --> G[表示を確認する]
    F --> G
